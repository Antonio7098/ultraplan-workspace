# Dimension 03.01: LLM Turn Loop Structure

## Purpose

Understand the basic model-call loop: prompt assembly, model call, output interpretation, action, repeat.

## Steps

1. Locate the model invocation loop.
2. Identify the loop body phases.
3. Identify turn boundaries.
4. Check whether each turn is persisted.
5. Check whether the loop is reusable across agents.

## Evidence to Capture

- Runner loop
- Turn records
- Model call wrapper
- Message assembly
- Final output condition

## Questions

1. What happens during one turn?
2. Does every turn call the model?
3. Can a turn include multiple tool calls?
4. Is turn state persisted?
5. Is the loop generic or agent-specific?

## Rating

| Score | Meaning |
|-------|---------|
| 1-3 | Absent, implicit, ad-hoc, or unsafe |
| 4-6 | Present but inconsistent, weakly documented, or fragile |
| 7-8 | Clear model with tests, explicit interfaces, and operational safeguards |
| 9-10 | Mature, durable, observable, extensible, and proven under failure or scale |
