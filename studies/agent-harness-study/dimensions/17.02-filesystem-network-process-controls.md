# Dimension 17.02: Filesystem, Network, and Process Controls

## Purpose

Study filesystem mounts, network policy, shell execution, process limits, and workspace cleanup.

## Steps

1. Identify filesystem access controls.
2. Check network policy enforcement.
3. Inspect shell command execution restrictions.
4. Check process limits (forks, memory, CPU).
5. Check workspace cleanup after execution.

## Evidence to Capture

- Filesystem mount config
- Network policy rules
- Shell execution wrappers
- Process limit configs
- Cleanup handlers

## Questions

1. What filesystem access does an agent have?
2. What network access does an agent have?
3. Can the agent spawn arbitrary processes?
4. Are resources cleaned up after execution?
