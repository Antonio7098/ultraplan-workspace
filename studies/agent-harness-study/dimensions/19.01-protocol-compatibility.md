# Dimension 19.01: Protocol Compatibility

## Purpose

Study support for MCP, OpenAPI, OTLP/OpenTelemetry, JSON Schema, provider APIs, and model-independent tool schemas.

## Steps

1. Identify protocols supported.
2. Check MCP (Model Context Protocol) support for tools/resources/prompts.
3. Check OpenTelemetry (OTLP) for trace export.
4. Check JSON Schema for tool definitions.
5. Check OpenAPI for API-driven tools.

## Evidence to Capture

- MCP client/server implementations
- OTLP exporters
- JSON Schema generators
- OpenAPI importers
- Protocol adapter code

## Questions

1. Which open protocols are supported?
2. Is MCP supported?
3. Is OpenTelemetry supported?
4. Are tool schemas portable across providers?

## Rating

| Score | Meaning |
|-------|---------|
| 1-3 | Absent, implicit, ad-hoc, or unsafe |
| 4-6 | Present but inconsistent, weakly documented, or fragile |
| 7-8 | Clear model with tests, explicit interfaces, and operational safeguards |
| 9-10 | Mature, durable, observable, extensible, and proven under failure or scale |

> Can external tools be added without writing custom adapters?
