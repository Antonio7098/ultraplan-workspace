# Dimension 14.03: Human Intervention and Takeover

## Purpose

Study whether humans can edit state, provide feedback, take over a sandbox, resume a run, or fork execution.

## Steps

1. Check whether humans can edit conversation state.
2. Check whether humans can inject feedback mid-run.
3. Inspect sandbox takeover capabilities.
4. Check whether humans can fork and continue runs.
5. Check whether human edits are tracked.

## Evidence to Capture

- State edit APIs
- Feedback injection points
- Sandbox takeover code
- Fork/resume APIs
- Edit audit trails

## Questions

1. Can humans edit agent state?
2. Can humans provide mid-run feedback?
3. Can humans take over execution?
4. Are human interventions traceable?

## Rating

| Score | Meaning |
|-------|---------|
| 1-3 | Absent, implicit, ad-hoc, or unsafe |
| 4-6 | Present but inconsistent, weakly documented, or fragile |
| 7-8 | Clear model with tests, explicit interfaces, and operational safeguards |
| 9-10 | Mature, durable, observable, extensible, and proven under failure or scale |

> Can a human correct the agent mid-flight without restarting?
