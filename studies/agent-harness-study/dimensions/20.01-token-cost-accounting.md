# Dimension 20.01: Token and Cost Accounting

## Purpose

Study whether token usage, model cost, tool cost, retry cost, and per-run cost are tracked.

## Steps

1. Identify token counting mechanisms.
2. Check whether model costs are calculated.
3. Check whether tool execution costs are tracked.
4. Inspect whether retries are factored into cost.
5. Check whether per-run cost summaries are available.

## Evidence to Capture

- Token counters
- Cost calculators
- Tool cost tracking
- Retry cost accounting
- Run cost summaries

## Questions

1. Are tokens counted per run?
2. Are costs attributed per model call?
3. Are tool execution costs tracked?
4. Are retry costs accounted for?
5. Are per-run cost summaries available?

## Rating

| Score | Meaning |
|-------|---------|
| 1-3 | Absent, implicit, ad-hoc, or unsafe |
| 4-6 | Present but inconsistent, weakly documented, or fragile |
| 7-8 | Clear model with tests, explicit interfaces, and operational safeguards |
| 9-10 | Mature, durable, observable, extensible, and proven under failure or scale |

> Can you answer 'what did this run cost?' in under a minute?
