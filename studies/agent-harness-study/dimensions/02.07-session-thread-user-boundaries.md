# Dimension 02.07: Session, Thread, and User Boundaries

## Purpose

Study how state is scoped across users, threads, projects, and runs.

## Steps

1. Identify primary IDs: user ID, thread ID, session ID, run ID, conversation ID, workspace ID, tenant ID.
2. Map which state belongs to each ID.
3. Check whether cross-user leakage is possible.
4. Inspect multi-tenant boundaries.
5. Check whether memory crosses threads intentionally.

## Evidence to Capture

- ID schemas
- Store queries
- Authorization filters
- Thread config
- Tenant isolation code
- Memory lookup rules

## Questions

1. What scopes state?
2. Can state leak across users or sessions?
3. Is cross-thread memory explicit?
4. Are tenant boundaries enforced at storage level?
5. Can users fork sessions safely?
