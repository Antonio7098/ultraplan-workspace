# Dimension 12.03: Prompt Evaluation and Experiments

## Purpose

Study datasets, prompt experiments, LLM-as-judge, regression tests, and model/prompt comparison.

## Steps

1. Identify evaluation datasets.
2. Check whether prompt changes are tested against expected outputs.
3. Inspect experiment tracking for A/B tests.
4. Check whether LLM-as-judge evaluation is used.
5. Check whether prompt regressions are caught in CI.

## Evidence to Capture

- Eval datasets
- Experiment trackers
- LLM-as-judge prompts
- Regression test suites
- CI prompt test integration

## Questions

1. Are prompt changes tested?
2. Are experiments tracked?
3. Is LLM-as-judge used for evaluation?
4. Are regressions caught before deployment?

## Rating

| Score | Meaning |
|-------|---------|
| 1-3 | Absent, implicit, ad-hoc, or unsafe |
| 4-6 | Present but inconsistent, weakly documented, or fragile |
| 7-8 | Clear model with tests, explicit interfaces, and operational safeguards |
| 9-10 | Mature, durable, observable, extensible, and proven under failure or scale |

> Can a prompt change be deployed with confidence that it won't regress?
