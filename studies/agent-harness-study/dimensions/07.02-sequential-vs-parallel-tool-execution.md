# Dimension 07.02: Sequential vs Parallel Tool Execution

## Purpose

Analyze when tools run one-at-a-time vs concurrently.

## Steps

1. Identify parallel execution support.
2. Determine whether tools declare side effects.
3. Check resource conflict detection.
4. Inspect result ordering.
5. Check whether parallel failures are isolated.
6. Identify configuration limits.

## Evidence to Capture

- Parallel executors
- Side-effect metadata
- Resource locks
- Semaphores
- Result collectors
- Parallel tests

## Questions

1. Can multiple tools run in one turn?
2. Which tools are safe to parallelize?
3. Are write tools serialized?
4. How are failures aggregated?
5. Is result order deterministic?

## Rating

| Score | Meaning |
|-------|---------|
| 1-3 | Absent, implicit, ad-hoc, or unsafe |
| 4-6 | Present but inconsistent, weakly documented, or fragile |
| 7-8 | Clear model with tests, explicit interfaces, and operational safeguards |
| 9-10 | Mature, durable, observable, extensible, and proven under failure or scale |

> Does concurrency improve latency without risking corruption?
