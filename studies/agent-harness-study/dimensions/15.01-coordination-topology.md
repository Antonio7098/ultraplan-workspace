# Dimension 15.01: Coordination Topology

## Purpose

Study whether coordination is supervisor-worker, peer-to-peer, group chat, graph, marketplace, or blackboard.

## Steps

1. Identify how agents communicate.
2. Classify the coordination topology.
3. Check whether topology is static or dynamic.
4. Inspect whether coordination is centralized or distributed.
5. Check whether agents can discover each other.

## Evidence to Capture

- Communication channels
- Topology definitions
- Agent discovery
- Central coordinator code
- Dynamic topology changes

## Questions

1. How do agents coordinate?
2. Is the topology fixed or dynamic?
3. Is there a single point of failure?
4. Can agents discover each other?

## Rating

| Score | Meaning |
|-------|---------|
| 1-3 | Absent, implicit, ad-hoc, or unsafe |
| 4-6 | Present but inconsistent, weakly documented, or fragile |
| 7-8 | Clear model with tests, explicit interfaces, and operational safeguards |
| 9-10 | Mature, durable, observable, extensible, and proven under failure or scale |
