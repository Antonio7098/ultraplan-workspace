# Dimension 17.01: Sandbox Boundary

## Purpose

Study where execution happens and what boundaries exist.

## Steps

1. Identify where tool code runs.
2. Classify execution environment: local process, container, remote sandbox, VM, browser sandbox, restricted process.
3. Check boundary enforcement mechanisms.
4. Check whether sandbox boundaries are configurable.

## Evidence to Capture

- Execution environment config
- Container/sandbox definitions
- Boundary enforcement code
- Sandbox configuration

## Questions

1. Where does code execute?
2. What boundaries exist between agents and the host?
3. Are boundaries enforced?
4. Can sandbox configuration be changed per-run?

## Rating

| Score | Meaning |
|-------|---------|
| 1-3 | Absent, implicit, ad-hoc, or unsafe |
| 4-6 | Present but inconsistent, weakly documented, or fragile |
| 7-8 | Clear model with tests, explicit interfaces, and operational safeguards |
| 9-10 | Mature, durable, observable, extensible, and proven under failure or scale |

> Can an agent escape its intended execution boundary?
