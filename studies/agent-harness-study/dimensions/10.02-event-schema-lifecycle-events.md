# Dimension 10.02: Event Schema and Lifecycle Events

## Purpose

Study whether lifecycle events are typed, ordered, timestamped, versioned, and linked to runtime objects.

## Steps

1. Identify event type definitions.
2. Check whether events are ordered (sequence numbers, timestamps).
3. Inspect event versioning strategy.
4. Check whether events link to parent IDs (run ID, span ID, tool call ID).
5. Check whether lifecycle events exist for creation, completion, failure, cancellation.

## Evidence to Capture

- Event schemas
- Event emitters
- Sequence numbers
- Event version fields
- Lifecycle event types

## Questions

1. Are events typed and versioned?
2. Are events ordered and timestamped?
3. Do events carry sufficient context?
4. Are lifecycle events comprehensive?

## Rating

| Score | Meaning |
|-------|---------|
| 1-3 | Absent, implicit, ad-hoc, or unsafe |
| 4-6 | Present but inconsistent, weakly documented, or fragile |
| 7-8 | Clear model with tests, explicit interfaces, and operational safeguards |
| 9-10 | Mature, durable, observable, extensible, and proven under failure or scale |

> Can you reconstruct the full lifecycle of any run from events alone?
