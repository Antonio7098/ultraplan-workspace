# Dimension 04.08: Agent-as-Tool and Workflow-as-Tool Composition

## Purpose

Study whether higher-level capabilities can be exposed as tools.

## Steps

1. Search for nested agents, workflows, skills, or subroutines exposed as tools.
2. Inspect input/output contracts.
3. Check whether nested execution is traced.
4. Check whether nested execution has separate limits.
5. Inspect cancellation and error propagation.
6. Check whether recursion is prevented.

## Evidence to Capture

- Agent tool wrappers
- Workflow tool wrappers
- Nested run APIs
- Child trace IDs
- Recursion guards
- Subagent result schemas

## Questions

1. Can one agent call another?
2. Are child runs bounded?
3. Are child run costs attributed?
4. Can nested tools recurse forever?
5. Does the parent receive structured results?

## Rating

| Score | Meaning |
|-------|---------|
| 1-3 | Absent, implicit, ad-hoc, or unsafe |
| 4-6 | Present but inconsistent, weakly documented, or fragile |
| 7-8 | Clear model with tests, explicit interfaces, and operational safeguards |
| 9-10 | Mature, durable, observable, extensible, and proven under failure or scale |

> Can composition scale without creating invisible nested loops?
