# Dimension 21.03: Extension Compatibility Testing

## Purpose

Study whether extension contracts have conformance tests, fixtures, examples, and stability guarantees.

## Steps

1. Check whether extension interfaces have conformance tests.
2. Inspect whether test fixtures exist for extension authors.
3. Check whether examples are provided.
4. Check whether stability guarantees are documented.
5. Check whether breaking changes are communicated.

## Evidence to Capture

- Conformance test suites
- Extension test fixtures
- Example implementations
- Stability documentation
- Breaking change policies

## Questions

1. Are extension contracts tested?
2. Are fixtures provided for extension authors?
3. Are examples provided?
4. Are stability guarantees documented?

## Rating

| Score | Meaning |
|-------|---------|
| 1-3 | Absent, implicit, ad-hoc, or unsafe |
| 4-6 | Present but inconsistent, weakly documented, or fragile |
| 7-8 | Clear model with tests, explicit interfaces, and operational safeguards |
| 9-10 | Mature, durable, observable, extensible, and proven under failure or scale |

> Can an extension author verify their implementation against the contract?

Write findings to `reports/repo/21.03-extension-compatibility-testing/{repo-name}.md`
