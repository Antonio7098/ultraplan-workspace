# Dimension 07.01: Tool Scheduling and Dispatch

## Purpose

Study how validated tool calls become running work.

## Steps

1. Locate dispatch logic.
2. Identify scheduling strategy: inline, async task, queue job, worker pool, remote server.
3. Check whether execution order is deterministic.
4. Inspect grouping/batching.
5. Check whether dispatch is traced.

## Evidence to Capture

- Dispatchers
- Executors
- Worker queues
- Tool batches
- Scheduling traces
- Execution config

## Questions

1. How does a tool call start?
2. Is tool execution inline or queued?
3. Are tool calls ordered?
4. Can tools be batched?
5. Is dispatch observable?

## Rating

| Score | Meaning |
|-------|---------|
