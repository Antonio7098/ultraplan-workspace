# Dimension 05.02: Working Memory and Scratchpad

## Purpose

Study whether the agent has temporary working memory separate from user-visible conversation.

## Steps

1. Search for scratchpad, notes, plan state, hidden state, or temporary variables.
2. Determine whether working memory is visible to the model, runtime, or user.
3. Check whether it persists across turns.
4. Check whether it is cleared at task boundaries.
5. Inspect whether it leaks sensitive content.

## Evidence to Capture

- Scratchpad fields
- Todo state
- Internal messages
- Hidden planner outputs
- Temporary state records

## Questions

1. Does the agent keep private task state?
2. Is it durable?
3. Is it exposed to users?
4. Does it pollute long-term memory?
5. Can it be audited?

## Rating

| Score | Meaning |
|-------|---------|
| 1-3 | Absent, implicit, ad-hoc, or unsafe |
| 4-6 | Present but inconsistent, weakly documented, or fragile |
| 7-8 | Clear model with tests, explicit interfaces, and operational safeguards |
| 9-10 | Mature, durable, observable, extensible, and proven under failure or scale |

> Can the agent think over multiple steps without confusing working notes with facts?
