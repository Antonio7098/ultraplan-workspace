# Dimension 09.04: Governance Evidence Generation

## Purpose

Study whether the system can produce artifacts showing what happened, what was checked, and what controls applied.

## Steps

1. Identify what evidence artifacts exist.
2. Check whether evidence includes policy decisions, approval timestamps, tool execution results.
3. Inspect whether evidence is machine-readable for SIEM integration.
4. Check whether evidence is retained according to policy.

## Evidence to Capture

- Evidence report generation
- Policy decision records
- Approval timestamps
- Tool execution logs
- SIEM integration code

## Questions

1. What evidence does the system produce?
2. Is evidence machine-readable?
3. Can evidence be reproduced?
4. Are evidence retention policies enforced?

## Rating

| Score | Meaning |
|-------|---------|
| 1-3 | Absent, implicit, ad-hoc, or unsafe |
| 4-6 | Present but inconsistent, weakly documented, or fragile |
| 7-8 | Clear model with tests, explicit interfaces, and operational safeguards |
| 9-10 | Mature, durable, observable, extensible, and proven under failure or scale |

> Can the system produce a compliance report for a given run?
