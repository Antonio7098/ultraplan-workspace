# Dimension 01.10: Replay and Determinism in Execution

## Purpose

Understand whether execution can be replayed and whether replay produces the same commands, state, or only the same audit trail.

## Steps

1. Identify replay mechanisms.
2. Separate deterministic workflow replay from non-deterministic LLM replay.
3. Check whether model calls are recorded or re-executed.
4. Identify what is treated as source of truth.
5. Look for replay tests.
6. Check how non-determinism is isolated.

## Evidence to Capture

- Event history
- Replay tests
- Recorded LLM responses
- Workflow commands
- Checkpoint lineage
- State reconstruction logic

## Questions

1. Can execution be replayed?
2. What exactly is replayed?
3. Are LLM calls replayed from logs or re-called?
4. Does replay verify correctness or only reconstruct state?
5. What non-determinism remains?
