# Dimension 03.06: Stuck and Doom-Loop Detection

## Purpose

Study whether the agent detects non-progress before hitting hard limits.

## Steps

1. Search for stuck detectors.
2. Identify repeated tool-call detection.
3. Identify repeated error detection.
4. Identify alternating patterns.
5. Identify no-progress monologues.
6. Check whether detection triggers stop, hint, compaction, escalation, or replanning.

## Evidence to Capture

- Stuck detector classes
- Recent-event windows
- Repeated-action thresholds
- Doom-loop constants
- Loop intervention handlers

## Questions

1. What stuck patterns are detected?
2. How far back does detection look?
3. Does compaction erase loop evidence?
4. What intervention happens?
5. Are false positives possible?
