# Dimension 06.01: Planning Location and Responsibility

## Purpose

Study where planning happens: inside prompts, runtime code, planner agent, workflow graph, or external system.

## Steps

1. Search for planner concepts.
2. Identify whether planning is implicit or explicit.
3. Determine whether planning is model-owned or runtime-owned.
4. Check whether planning is optional.
5. Identify planning boundaries.

## Evidence to Capture

- Planner prompts
- Planning agents
- Workflow graphs
- Task decomposition code
- Planning config

## Questions

1. Where does planning happen?
2. Who owns the plan?
3. Is planning required?
4. Is planning visible?
5. Is planning reusable?

## Rating

| Score | Meaning |
|-------|---------|
| 1-3 | Absent, implicit, ad-hoc, or unsafe |
| 4-6 | Present but inconsistent, weakly documented, or fragile |
| 7-8 | Clear model with tests, explicit interfaces, and operational safeguards |
| 9-10 | Mature, durable, observable, extensible, and proven under failure or scale |

> Is planning a real runtime object or just prose in a prompt?

## Output

Write findings to `reports/repo/06.01-planning-location-and-responsibility/{repo-name}.md`
