# Dimension 03.05: Reflection, ReAsk, and Self-Correction Loops

## Purpose

Analyze how the agent responds to malformed outputs, failed validation, failed tests, or weak answers.

## Steps

1. Identify validation failure paths.
2. Identify reask/reflection prompts.
3. Check retry budgets.
4. Determine whether reflection sees enough evidence.
5. Inspect whether repeated reflection can loop forever.
6. Check whether self-correction is evaluated.

## Evidence to Capture

- Reask loops
- Reflection prompts
- Validation errors
- Retry counters
- Critic agents
- Test feedback loops

## Questions

1. When does the agent self-correct?
2. Is correction bounded?
3. What evidence is shown during correction?
4. Does correction improve outputs or hide errors?
5. Are repeated failures escalated?

## Rating

| Score | Meaning |

> Can the system recover from bad model output without hallucinating success?
