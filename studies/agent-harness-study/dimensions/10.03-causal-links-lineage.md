# Dimension 10.03: Causal Links and Lineage

## Purpose

Study whether outputs can be traced back to prompts, tools, retrieved context, memory, approvals, artifacts, and model versions.

## Steps

1. Check whether model outputs are linked to input prompts.
2. Check whether tool results are linked to tool definitions and arguments.
3. Check whether retrieved context includes source provenance.
4. Check whether approvals are linked to actions.
5. Check whether artifacts are linked to the runs that produced them.

## Evidence to Capture

- Lineage metadata
- Input/output linking
- Provenance fields
- Approval-action links
- Artifact-run associations

## Questions

1. Can every output be traced to its inputs?
2. Is provenance preserved through transformations?
3. Are model versions tracked in lineage?
4. Can causal chains be audited?

## Rating

| Score | Meaning |
|-------|---------|
| 1-3 | Absent, implicit, ad-hoc, or unsafe |
| 4-6 | Present but inconsistent, weakly documented, or fragile |
| 7-8 | Clear model with tests, explicit interfaces, and operational safeguards |
| 9-10 | Mature, durable, observable, extensible, and proven under failure or scale |

> Can you trace a final answer back to the specific tool call that provided each fact?
