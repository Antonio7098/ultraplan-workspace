# Dimension 05.06: Memory Compression and Summarization

## Purpose

Analyze how long histories are compressed into durable summaries.

## Steps

1. Identify summarization triggers.
2. Inspect summary prompts.
3. Inspect coverage tracking.
4. Check whether summaries replace or supplement raw history.
5. Check whether summaries are evaluated.
6. Inspect failure behavior.

## Evidence to Capture

- Summary records
- Token budget logic
- Summary prompts
- Coverage ranges
- Raw-to-summary links
- Summary refresh logic

## Questions

1. When does summarization happen?
2. What evidence does the summary cover?
3. Can summary drift be detected?
4. Is raw history retained?
5. Can summaries be regenerated?

| Score | Meaning |
