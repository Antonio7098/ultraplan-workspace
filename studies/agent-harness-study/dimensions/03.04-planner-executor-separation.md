# Dimension 03.04: Planner/Executor Separation

## Purpose

Study whether planning is separate from execution or embedded in the model loop.

## Steps

1. Search for planner agents, todo systems, task lists, or plan phases.
2. Determine whether plans are persisted.
3. Identify who updates the plan.
4. Check whether execution is constrained by the plan.
5. Inspect replanning behavior after failure.

## Evidence to Capture

- Planner prompts
- Todo/task records
- Plan schemas
- Plan-update tools
- Executor loops
- Replanning triggers

## Questions

1. Is there an explicit plan?
2. Does the plan control execution?
3. Can the plan change?
4. Are plan changes traceable?
5. Is planning useful or decorative?

## Rating

| Score | Meaning |
|-------|---------|
| 1-3 | Absent, implicit, ad-hoc, or unsafe |
| 4-6 | Present but inconsistent, weakly documented, or fragile |
| 7-8 | Clear model with tests, explicit interfaces, and operational safeguards |
| 9-10 | Mature, durable, observable, extensible, and proven under failure or scale |

> Can the system compare what it planned to what it actually did?
