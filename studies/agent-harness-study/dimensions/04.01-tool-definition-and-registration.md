# Dimension 04.01: Tool Definition and Registration

## Purpose

Study how tools are declared, named, grouped, registered, and exposed to agents.

## Steps

1. Identify tool declaration patterns.
2. Classify registration style: class-based, function-based, decorator-based, factory-based, static list, MCP server.
3. Inspect naming conventions.
4. Identify tool namespaces/catalogs.
5. Check whether registration is dynamic.

## Evidence to Capture

- Tool base classes
- Decorators
- Registries
- Catalog definitions
- Namespaces
- MCP adapters

## Questions

1. How does a new tool enter the system?
2. Is tool registration explicit?
3. Can tools be discovered dynamically?
4. Are names stable?
5. Can multiple agents see different tool sets?

## Rating

| Score | Meaning |
|-------|---------|
| 1-3 | Absent, implicit, ad-hoc, or unsafe |
| 4-6 | Present but inconsistent, weakly documented, or fragile |
| 7-8 | Clear model with tests, explicit interfaces, and operational safeguards |
| 9-10 | Mature, durable, observable, extensible, and proven under failure or scale |

> Can you add a tool without touching unrelated runtime code?
