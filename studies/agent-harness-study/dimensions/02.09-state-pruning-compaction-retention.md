# Dimension 02.09: State Pruning, Compaction, and Retention

## Purpose

Analyze how the system prevents state/history growth from becoming unbounded.

## Steps

1. Identify pruning policies.
2. Inspect conversation summarization.
3. Inspect event/checkpoint garbage collection.
4. Check retention configuration.
5. Determine whether compaction preserves replay/debugging.
6. Identify privacy deletion behavior.

## Evidence to Capture

- Retention configs
- Summary records
- Checkpoint pruning
- Event compaction
- Token budget pruning
- Delete/anonymize paths

## Questions

1. What grows forever?
2. What gets summarized?
3. What gets deleted?
4. Does compaction break replay?
5. Can users request deletion?
