# Dimension 22.03: Docs, Examples, and Contributor Workflow

## Purpose

Study whether the repo teaches contributors how to add agents, tools, evals, memory, tracing, and policies safely.

## Steps

1. Check whether contribution guides exist.
2. Inspect whether examples cover all extension types.
3. Check whether API docs are generated.
4. Check whether there are tutorials for common extensions.
5. Check whether there are template/starter repos.

## Evidence to Capture

- Contribution guides
- Example directories
- API documentation
- Tutorials
- Template projects

## Questions

1. Are contribution guides clear?
2. Are examples comprehensive?
3. Is API documentation available?
4. Are there tutorials for key tasks?

## Rating

| Score | Meaning |
|-------|---------|
| 1-3 | Absent, implicit, ad-hoc, or unsafe |
| 4-6 | Present but inconsistent, weakly documented, or fragile |
| 7-8 | Clear model with tests, explicit interfaces, and operational safeguards |
| 9-10 | Mature, durable, observable, extensible, and proven under failure or scale |

> Can a new contributor add a tool in under an hour without asking for help?

## Output

Write findings to `reports/repo/22.03-docs-examples-and-contributor-workflow/{repo-name}.md`
