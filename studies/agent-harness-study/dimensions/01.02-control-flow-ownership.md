# Dimension 01.02: Control-Flow Ownership

## Purpose

Understand who owns control flow: the framework, the model, the graph, the user, the scheduler, or the tool runtime.

## Steps

1. Find the object/function that decides the next step.
2. Identify whether the LLM can choose the next action directly.
3. Identify whether the runtime can override, pause, reroute, or terminate the model.
4. Look for explicit next-step enums, state machines, or transition functions.
5. Compare model-driven control vs runtime-driven control.

## Evidence to Capture

- NextStep types
- State transition functions
- Router functions
- Tool-call dispatch logic
- Handoff logic
- Termination evaluators

## Questions

1. Who decides what happens next?
2. Can the LLM bypass runtime control?
3. Can the runtime reject or rewrite the next action?
4. Are transitions explicit or scattered?
5. Is control flow testable without calling an LLM?

## Rating

| Score | Meaning |
|-------|---------|
| 1-3 | Absent, implicit, ad-hoc, or unsafe |
| 4-6 | Present but inconsistent, weakly documented, or fragile |
| 7-8 | Clear model with tests, explicit interfaces, and operational safeguards |
| 9-10 | Mature, durable, observable, extensible, and proven under failure or scale |

> If the model asks for a dangerous or invalid next step, does the runtime have authority to stop it?
