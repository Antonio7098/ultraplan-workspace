# Dimension 04.02: Tool Schema Generation and Validation

## Purpose

Analyze how tool argument schemas are generated, validated, and shown to the model.

## Steps

1. Identify schema source: Pydantic, Zod, JSON Schema, type annotations, manual schema.
2. Inspect validation before execution.
3. Check whether schemas include descriptions.
4. Check whether invalid arguments trigger correction.
5. Inspect schema compatibility with model providers.

## Evidence to Capture

- Schema generators
- Validator code
- Function signatures
- JSON Schema output
- Invalid argument tests

## Questions

1. Are schemas generated or handwritten?
2. Are descriptions useful?
3. Is validation strict?
4. Are defaults handled clearly?
5. Are schemas portable across providers?

## Rating

| Score | Meaning |
|-------|---------|
| 1-3 | Absent, implicit, ad-hoc, or unsafe |
| 4-6 | Present but inconsistent, weakly documented, or fragile |
| 7-8 | Clear model with tests, explicit interfaces, and operational safeguards |
| 9-10 | Mature, durable, observable, extensible, and proven under failure or scale |

> Would the model understand how to call this tool correctly?
