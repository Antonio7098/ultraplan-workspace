# Dimension 21.02: Provider and Backend Adapters

## Purpose

Study whether models, stores, vector DBs, tracing sinks, queues, and sandboxes are swappable.

## Steps

1. Identify swappable backend interfaces.
2. Check which backends have multiple implementations.
3. Inspect adapter pattern usage.
4. Check whether adapters are interchangeable at runtime.
5. Check whether adapter configuration is externalized.

## Evidence to Capture

- Backend abstractions
- Provider implementations
- Adapter factories
- Runtime backend selection
- Adapter configs

## Questions

1. Are backends swappable?
2. Which backends have multiple implementations?
3. Can backends be swapped at runtime?
4. Are adapter implementations tested?

## Rating

| Score | Meaning |
|-------|---------|
| 1-3 | Absent, implicit, ad-hoc, or unsafe |
| 4-6 | Present but inconsistent, weakly documented, or fragile |
| 7-8 | Clear model with tests, explicit interfaces, and operational safeguards |
| 9-10 | Mature, durable, observable, extensible, and proven under failure or scale |
