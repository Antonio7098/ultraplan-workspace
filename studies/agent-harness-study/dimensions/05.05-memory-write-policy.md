# Dimension 05.05: Memory Write Policy

## Purpose

Study when the system decides to store new memory.

## Steps

1. Search for memory extraction or write triggers.
2. Identify automatic vs explicit writes.
3. Check validation before storing.
4. Inspect conflict resolution.
5. Check whether users can approve memory writes.
6. Check whether memory has confidence metadata.

## Evidence to Capture

- Memory write functions
- Extractors
- Approval flows
- Conflict handlers
- Confidence scores
- Memory update events

## Questions

1. What causes memory to be written?
2. Can the model write arbitrary memory?
3. Are facts verified?
4. Can users correct memory?
5. Are sensitive facts excluded?

## Rating

| Score | Meaning |
|-------|---------|
| 1-3 | Absent, implicit, ad-hoc, or unsafe |
| 4-6 | Present but inconsistent, weakly documented, or fragile |
| 7-8 | Clear model with tests, explicit interfaces, and operational safeguards |
| 9-10 | Mature, durable, observable, extensible, and proven under failure or scale |

> Does the system remember deliberately rather than accidentally?
