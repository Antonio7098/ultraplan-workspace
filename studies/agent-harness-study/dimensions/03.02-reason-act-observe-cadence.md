# Dimension 03.02: Reason/Act/Observe Cadence

## Purpose

Study whether the system follows ReAct-style reasoning, tool execution, and observation feedback.

## Steps

1. Identify model output parsing.
2. Identify action/tool-call extraction.
3. Identify observation injection.
4. Check whether the model sees raw or summarized tool results.
5. Determine whether reasoning is hidden, structured, or stored.

## Evidence to Capture

- Tool call messages
- Observation messages
- Scratchpad prompts
- ReAct parser
- Tool result formatting

## Questions

1. How does the model decide actions?
2. How are observations returned?
3. Is reasoning visible or hidden?
4. Are failed observations fed back?
5. Can observations be compacted?

## Rating

| Score | Meaning |
|-------|---------|
| 1-3 | Absent, implicit, ad-hoc, or unsafe |
| 4-6 | Present but inconsistent, weakly documented, or fragile |
| 7-8 | Clear model with tests, explicit interfaces, and operational safeguards |
| 9-10 | Mature, durable, observable, extensible, and proven under failure or scale |

> Does the loop teach the model what happened after every action?
