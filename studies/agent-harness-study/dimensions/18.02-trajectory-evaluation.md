# Dimension 18.02: Trajectory Evaluation

## Purpose

Study whether the system evaluates intermediate reasoning, tool choices, context usage, and recovery, not just final output.

## Steps

1. Check whether evals assess intermediate steps.
2. Inspect whether tool choice is evaluated.
3. Check whether context usage is measured.
4. Check whether recovery from failure is evaluated.
5. Check whether trajectory scores are tracked.

## Evidence to Capture

- Step-by-step eval code
- Tool choice evaluators
- Context usage metrics
- Recovery eval cases
- Trajectory scoring

## Questions

1. Are intermediate steps evaluated?
2. Is tool selection quality measured?
3. Is context usage evaluated?
4. Is recovery behavior measured?

## Rating

| Score | Meaning |
|-------|---------|
| 1-3 | Absent, implicit, ad-hoc, or unsafe |
| 4-6 | Present but inconsistent, weakly documented, or fragile |
| 7-8 | Clear model with tests, explicit interfaces, and operational safeguards |
| 9-10 | Mature, durable, observable, extensible, and proven under failure or scale |

> Can you tell whether the agent took a good path even if the final answer was wrong?
