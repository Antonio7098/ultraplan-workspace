# Dimension 06.05: Objective and Progress Tracking

## Purpose

Analyze how systems know whether they are getting closer to the goal.

## Steps

1. Identify goal representation.
2. Inspect progress markers.
3. Check whether progress is measured by tests, tool success, model judgement, or user approval.
4. Inspect completion criteria.
5. Check whether progress is shown in traces/UI.

## Evidence to Capture

- Goal objects
- Progress events
- Test results
- Milestone records
- Success criteria
- UI status

## Questions

1. What is the goal?
2. How is progress measured?
3. Can the model fake progress?
4. Are blockers recorded?
5. Is final success independently checked?

## Rating

| Score | Meaning |
|-------|---------|
| 1-3 | Absent, implicit, ad-hoc, or unsafe |
| 4-6 | Present but inconsistent, weakly documented, or fragile |
| 7-8 | Clear model with tests, explicit interfaces, and operational safeguards |
| 9-10 | Mature, durable, observable, extensible, and proven under failure or scale |

> Does the agent know the difference between activity and progress?
