# Dimension 01.07: Concurrency and Parallel Advancement

## Purpose

Study whether multiple steps, tools, agents, or workflows can advance at once.

## Steps

1. Identify async usage, threads, workers, fibers, processes, or queues.
2. Determine whether concurrency happens inside one run or across many runs.
3. Identify resource locks.
4. Identify ordering guarantees.
5. Check how parallel failures are collected.
6. Check whether partial parallel success is preserved.

## Evidence to Capture

- asyncio.gather
- Thread pools
- Worker concurrency
- Semaphores
- Resource lock managers
- Parallel node execution
- Tool execution batches

## Questions

1. What can run in parallel?
2. What must be serialized?
3. How are shared resources protected?
4. Are results ordered deterministically?
5. What happens when one parallel branch fails?

## Rating

| Score | Meaning |
|-------|---------|
| 1-3 | Absent, implicit, ad-hoc, or unsafe |
| 4-6 | Present but inconsistent, weakly documented, or fragile |
| 7-8 | Clear model with tests, explicit interfaces, and operational safeguards |
| 9-10 | Mature, durable, observable, extensible, and proven under failure or scale |

> Can read-only work run fast without making stateful work unsafe?
