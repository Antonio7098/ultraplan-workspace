# Dimension 16.02: Diff, Review, and Rollback

## Purpose

Study whether artifacts can be compared, reverted, approved, commented on, and associated with runs.

## Steps

1. Check whether artifacts support diff/comparison.
2. Inspect review workflow for artifacts.
3. Check rollback support.
4. Check whether comments/annotations are supported.
5. Check whether artifact changes are linked to runs.

## Evidence to Capture

- Diff generators
- Review workflow code
- Rollback handlers
- Comment/annotation models
- Artifact change logs

## Questions

1. Can artifacts be compared?
2. Is there a review workflow?
3. Can artifacts be rolled back?
4. Are artifact changes traceable to runs?

## Rating

| Score | Meaning |
|-------|---------|
| 1-3 | Absent, implicit, ad-hoc, or unsafe |
| 4-6 | Present but inconsistent, weakly documented, or fragile |
| 7-8 | Clear model with tests, explicit interfaces, and operational safeguards |
| 9-10 | Mature, durable, observable, extensible, and proven under failure or scale |

> Can a bad artifact change be reverted with full audit trail?

## Output

Write findings to `reports/repo/16.02-diff-review-and-rollback/{repo-name}.md`
