# Dimension 06.07: Plan Observability and Evaluation

## Purpose

Analyze how planning quality is traced and evaluated.

## Steps

1. Check whether plans are logged.
2. Check whether plan execution is linked to plan items.
3. Identify plan quality metrics.
4. Inspect eval datasets for planning.
5. Check whether planning regressions are tested.

## Evidence to Capture

- Plan traces
- Plan item IDs
- Execution links
- Eval harnesses
- Planning metrics
- Regression tests

## Questions

1. Are plans observable?
2. Can each action be linked to a plan item?
3. Are plans evaluated?
4. Can poor planning be diagnosed?
5. Does planning improve success rate?

## Rating

| Score | Meaning |
|-------|---------|
| 1-3 | Absent, implicit, ad-hoc, or unsafe |
| 4-6 | Present but inconsistent, weakly documented, or fragile |
| 7-8 | Clear model with tests, explicit interfaces, and operational safeguards |
| 9-10 | Mature, durable, observable, extensible, and proven under failure or scale |

> Can you debug a failed run by comparing plan vs execution?
