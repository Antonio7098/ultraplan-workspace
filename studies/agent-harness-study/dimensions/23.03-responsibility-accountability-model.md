# Dimension 23.03: Responsibility and Accountability Model

## Purpose

Study whether responsibility belongs to the model, runtime, tool author, user, operator, or organization.

## Steps

1. Identify where responsibility is assigned in the system.
2. Check whether model outputs carry disclaimers or attribution.
3. Inspect whether tool execution is attributed.
4. Check whether human decisions are recorded.
5. Check whether accountability is documented.

## Evidence to Capture

- Policy attribution
- Output disclaimers
- Tool attribution records
- Human decision logs
- Accountability documentation

## Questions

1. Who is responsible for agent actions?
2. Is model output attributed?
3. Are tool decisions attributed?
4. Are human approvals recorded?
5. Is accountability documented?

## Rating

| Score | Meaning |
|-------|---------|
| 1-3 | Absent, implicit, ad-hoc, or unsafe |
| 4-6 | Present but inconsistent, weakly documented, or fragile |
| 7-8 | Clear model with tests, explicit interfaces, and operational safeguards |
| 9-10 | Mature, durable, observable, extensible, and proven under failure or scale |

> Can the system answer 'who is responsible for this action?'
