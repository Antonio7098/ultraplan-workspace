# Dimension 19.02: Portable Trace, Eval, and Prompt Schemas

## Purpose

Study whether traces, evals, prompts, and tool calls can move between platforms.

## Steps

1. Check whether trace format is provider-agnostic.
2. Check whether eval datasets are portable.
3. Check whether prompt templates use standard formats.
4. Check whether tool schemas are portable across model providers.
5. Check whether data migration between platforms is supported.

## Evidence to Capture

- Trace format abstraction
- Dataset portability
- Prompt template portability
- Cross-provider schema compatibility
- Export/import tools

## Questions

1. Can traces be moved between platforms?
2. Can eval datasets be reused across systems?
3. Can prompts be migrated?
4. Are tool schemas provider-independent?

## Rating

| Score | Meaning |
|-------|---------|
| 1-3 | Absent, implicit, ad-hoc, or unsafe |
| 4-6 | Present but inconsistent, weakly documented, or fragile |
| 7-8 | Clear model with tests, explicit interfaces, and operational safeguards |
| 9-10 | Mature, durable, observable, extensible, and proven under failure or scale |

> Can you migrate from one tracing platform to another without rewriting data?
