# Dimension 02.01: State Taxonomy and Ownership

## Purpose

Classify every kind of state and identify who owns it.

## Steps

1. List all state categories: conversation state, execution state, tool state, memory state, artifact state, approval state, eval state, runtime configuration.
2. Identify where each state type lives.
3. Identify whether each state type is durable or ephemeral.
4. Identify the owner module.
5. Identify whether state crosses process/session boundaries.

## Evidence to Capture

- State classes
- Database models
- In-memory stores
- Context objects
- Session records
- Event records

## Questions

1. What kinds of state exist?
2. Which state is source-of-truth?
3. Which state is derived?
4. Which state is ephemeral?
5. Which state is safe to lose?

## Rating

| Score | Meaning |
|-------|---------|
| 1-3 | Absent, implicit, ad-hoc, or unsafe |
| 4-6 | Present but inconsistent, weakly documented, or fragile |
| 7-8 | Clear model with tests, explicit interfaces, and operational safeguards |
| 9-10 | Mature, durable, observable, extensible, and proven under failure or scale |

> Can the repo explain which state must survive a crash?
