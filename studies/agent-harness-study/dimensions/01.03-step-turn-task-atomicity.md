# Dimension 01.03: Step, Turn, and Task Atomicity

## Purpose

Define the smallest meaningful unit of execution.

## Steps

1. Identify the system's vocabulary: step, turn, message, action, observation, tool call, workflow task, node execution.
2. Determine which unit is persisted.
3. Determine which unit is retried.
4. Determine which unit is traced.
5. Determine which unit is shown to the user.

## Evidence to Capture

- Step classes
- Turn records
- Event schemas
- Tool call records
- Checkpoint boundaries
- Retry wrappers
- Trace/span hierarchy

## Questions

1. What is the atomic unit of execution?
2. Is the atomic unit the same for persistence, tracing, retry, and UI?
3. Can partially completed steps exist?
4. What happens if a crash occurs mid-step?
5. Are tool calls their own atomic units?
