# Dimension 08.01: Capability Model and Trust Boundaries

## Purpose

Identify what the agent is allowed to access and where trust boundaries sit.

## Steps

1. Inventory capabilities the system exposes.
2. Map trust boundaries between agent, user, runtime, tools, and external systems.
3. Examine agent/user/system privilege separation.
4. Distinguish runtime authority from tool authority.

## Evidence to Capture

- Tool permissions
- Auth checks
- Sandbox rules
- Secrets access
- Network policy

## Questions

1. What can the agent do?
2. What can the model only request but not directly do?
3. Where is authority enforced?
4. Are dangerous capabilities isolated?

## Rating

| Score | Meaning |
|-------|---------|
| 1-3 | Absent, implicit, ad-hoc, or unsafe |
| 4-6 | Present but inconsistent, weakly documented, or fragile |
| 7-8 | Clear model with tests, explicit interfaces, and operational safeguards |
| 9-10 | Mature, durable, observable, extensible, and proven under failure or scale |

> Can the model request power without possessing power?
