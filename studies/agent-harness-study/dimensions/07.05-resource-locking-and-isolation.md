# Dimension 07.05: Resource Locking and Isolation

## Purpose

Study how shared resources are protected during tool execution.

## Steps

1. Identify resources: files, shell, browser, database, network, workspace, secrets.
2. Inspect lock managers.
3. Check lock granularity.
4. Check deadlock prevention.
5. Inspect sandbox boundaries.

## Evidence to Capture

- Resource lock manager
- Workspace locks
- File locks
- Database transactions
- Sandbox config
- Deadlock handling

## Questions

1. Which resources are shared?
2. What protects them?
3. Are locks coarse or fine-grained?
4. Can deadlocks occur?
5. Are resource conflicts visible?
