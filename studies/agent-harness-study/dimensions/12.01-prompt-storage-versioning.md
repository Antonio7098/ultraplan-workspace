# Dimension 12.01: Prompt Storage and Versioning

## Purpose

Study whether prompts live in code, files, database, prompt registry, or external platform, and whether versions are recorded at runtime.

## Steps

1. Identify where prompt templates are stored.
2. Check whether prompts are versioned.
3. Inspect how prompt versions are associated with runs.
4. Check whether prompts can be deployed independently of code.

## Evidence to Capture

- Prompt file locations
- Version identifiers
- Run-to-prompt associations
- Prompt deployment mechanisms

## Questions

1. Where are prompts stored?
2. Are prompt versions tracked?
3. Can a run be traced to the exact prompt version used?
4. Can prompts be updated without redeploying code?

## Rating

| Score | Meaning |
|-------|---------|
| 1-3 | Absent, implicit, ad-hoc, or unsafe |
| 4-6 | Present but inconsistent, weakly documented, or fragile |
| 7-8 | Clear model with tests, explicit interfaces, and operational safeguards |
| 9-10 | Mature, durable, observable, extensible, and proven under failure or scale |

> Can you tell exactly which prompt version produced a given output?
