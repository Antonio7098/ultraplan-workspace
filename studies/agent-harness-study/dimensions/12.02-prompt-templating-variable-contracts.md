# Dimension 12.02: Prompt Templating and Variable Contracts

## Purpose

Study template engines, required variables, validation, escaping, and missing-variable behavior.

## Steps

1. Identify template engine used.
2. Inspect variable injection points.
3. Check whether required variables are validated.
4. Check escaping and injection prevention.
5. Check behavior when variables are missing.

## Evidence to Capture

- Template engine / library
- Variable schemas
- Validation code
- Escaping utilities
- Missing variable handlers

## Questions

1. How are prompts parameterized?
2. Are variable contracts explicit?
3. Is missing-variable behavior predictable?
4. Are variables properly escaped?

## Rating

| Score | Meaning |
|-------|---------|
| 1-3 | Absent, implicit, ad-hoc, or unsafe |
| 4-6 | Present but inconsistent, weakly documented, or fragile |
| 7-8 | Clear model with tests, explicit interfaces, and operational safeguards |
| 9-10 | Mature, durable, observable, extensible, and proven under failure or scale |
