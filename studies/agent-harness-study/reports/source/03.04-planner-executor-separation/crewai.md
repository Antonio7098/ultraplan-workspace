## Summary

The source contains two planning surfaces. The modern per-agent path uses structured planner output from `AgentReasoning` (`studies/agent-harness-study/sources/crewai/lib/crewai/src/crewai/utilities/reasoning_handler.py:30-47`, `:107-185`), converts it into typed todos, and executes those todos through a flow-based `AgentExecutor` (`studies/agent-harness-study/sources/crewai/lib/crewai/src/crewai/experimental/agent_executor.py:126-161`, `:314-372`) and per-step `StepExecutor` (`studies/agent-harness-study/sources/crewai/lib/crewai/src/crewai/agents/step_executor.py:63-76`). The legacy crew-level `CrewPlanner` (`studies/agent-harness-study/sources/crewai/lib/crewai/src/crewai/utilities/planning_handler.py:15-164`) instead injects a free-form plan into task descriptions (`studies/agent-harness-study/sources/crewai/lib/crewai/src/crewai/crew.py:1427-1453`).

The modern path is a clear planner/executor separation: plans are typed, owned by executor state, constrained by dependency and iteration guards, observed after execution, and replanned through bounded flow transitions. Plan changes are observable through typed events, execution logs, telemetry, and output fields. The main limitations are that the agent has no direct plan-update tool, some replan detection is heuristic, default persistence is in process unless checkpointing is configured, and observer failure conservatively continues execution.

## Rating

**8/10 — Clear model with tests, explicit interfaces, and operational safeguards.**

The modern path exceeds a merely present or decorative planner: it has structured plan and todo schemas, explicit planner and executor phases, dependency-aware execution, bounded replanning, failure handling, event telemetry, and broad tests. It falls short of the mature 9–10 range because there is no plan-specific durable store, no tested step-level crash-resume path, no LLM-callable plan mutation interface, and some decisions rely on message-string heuristics or optimistic observer-failure behavior (`studies/agent-harness-study/sources/crewai/lib/crewai/src/crewai/experimental/agent_executor.py:2452-2502`, `studies/agent-harness-study/sources/crewai/lib/crewai/src/crewai/agents/planner_observer.py:191-213`). The legacy crew-level path is materially weaker because its plan is free-form and injected into task text (`studies/agent-harness-study/sources/crewai/lib/crewai/src/crewai/utilities/planning_handler.py:15-164`, `studies/agent-harness-study/sources/crewai/lib/crewai/src/crewai/crew.py:1448`).

## Answers to Dimension Questions

1. **Is there an explicit plan?** Yes. The modern planner emits a validated structured plan, and runtime todos add status, result, and dependency state (`studies/agent-harness-study/sources/crewai/lib/crewai/src/crewai/utilities/planning_types.py:14-145`; `studies/agent-harness-study/sources/crewai/lib/crewai/src/crewai/utilities/reasoning_handler.py:50-104`). The legacy crew-level plan is explicit but free-form (`studies/agent-harness-study/sources/crewai/lib/crewai/src/crewai/utilities/planning_handler.py:15-164`).

2. **Does the plan control execution?** Yes in the modern path. Todo readiness and dependency checks control scheduling (`studies/agent-harness-study/sources/crewai/lib/crewai/src/crewai/utilities/planning_types.py:112-145`), the executor routes between sequential and parallel todo execution (`studies/agent-harness-study/sources/crewai/lib/crewai/src/crewai/experimental/agent_executor.py:1033-1185`), and a configured required tool is enforced at step execution (`studies/agent-harness-study/sources/crewai/lib/crewai/src/crewai/agents/step_executor.py:504-526`). The legacy plan is injected into task text, so the inspected implementation provides weaker evidence that it constrains later execution (`studies/agent-harness-study/sources/crewai/lib/crewai/src/crewai/crew.py:1448`).

3. **Can the plan change?** Yes. Pre-execution refinement can modify the plan (`studies/agent-harness-study/sources/crewai/lib/crewai/src/crewai/utilities/reasoning_handler.py:278-349`), and runtime replanning replaces pending todos while preserving completed, failed, and running history (`studies/agent-harness-study/sources/crewai/lib/crewai/src/crewai/experimental/agent_executor.py:2504-2589`; `studies/agent-harness-study/sources/crewai/lib/crewai/src/crewai/utilities/planning_types.py:185-195`). No LLM-callable plan-update tool was found in the inspected planner schema or executor paths; changes are flow-driven (`studies/agent-harness-study/sources/crewai/lib/crewai/src/crewai/utilities/reasoning_handler.py:50-104`; `studies/agent-harness-study/sources/crewai/lib/crewai/src/crewai/experimental/agent_executor.py:897-1016`).

4. **Are plan changes traceable?** Largely yes. Replan and refinement transitions emit typed events, update replan state, record reasons, and appear in execution logs and output (`studies/agent-harness-study/sources/crewai/lib/crewai/src/crewai/events/types/observation_events.py:98-131`; `studies/agent-harness-study/sources/crewai/lib/crewai/src/crewai/experimental/agent_executor.py:643-655`, `:922-1010`; `studies/agent-harness-study/sources/crewai/lib/crewai/src/crewai/lite_agent_output.py:43-59`). The inspected tests do not demonstrate crash recovery for a partially completed plan, so durable cross-process traceability has a gap.

5. **Is planning useful or decorative?** The modern planner is useful rather than decorative: it controls dependency-aware scheduling, required-tool validation, step limits, observations, replanning, and final state (`studies/agent-harness-study/sources/crewai/lib/crewai/src/crewai/utilities/planning_types.py:112-145`; `studies/agent-harness-study/sources/crewai/lib/crewai/src/crewai/agents/step_executor.py:504-526`; `studies/agent-harness-study/sources/crewai/lib/crewai/src/crewai/experimental/agent_executor.py:2452-2589`). The legacy crew-level planner is less operationally strong because the inspected path only injects free-form text into a task description (`studies/agent-harness-study/sources/crewai/lib/crewai/src/crewai/crew.py:1427-1453`).

## Architectural Decisions

- **Typed planner output and typed runtime state.** The planner's function-call schema is closed and explicit, while runtime todo state adds lifecycle and dependency information (`studies/agent-harness-study/sources/crewai/lib/crewai/src/crewai/utilities/reasoning_handler.py:50-104`; `studies/agent-harness-study/sources/crewai/lib/crewai/src/crewai/utilities/planning_types.py:27-145`).
- **Executor-owned plan mutation.** Plan generation, refinement, replan, and reset are centralized in the executor flow rather than exposed as arbitrary agent actions (`studies/agent-harness-study/sources/crewai/lib/crewai/src/crewai/experimental/agent_executor.py:314-372`, `:897-1016`, `:2504-2589`, `:2746-2760`). This protects plan state but reduces direct agent control.
- **Observation-driven continuation.** The executor uses reasoning effort to select whether to skip observation, continue, refine, or replan after a step (`studies/agent-harness-study/sources/crewai/lib/crewai/src/crewai/experimental/agent_executor.py:608-895`).
- **Bounded adaptation.** Refinement, replanning, step iterations, timeouts, and total planning steps have explicit limits (`studies/agent-harness-study/sources/crewai/lib/crewai/src/crewai/agent/planning_config.py:98-142`; `studies/agent-harness-study/sources/crewai/lib/crewai/src/crewai/utilities/reasoning_handler.py:278-349`).
- **Event-first operational visibility.** Planning lifecycle events are emitted separately from ordinary execution and are consumed by console, telemetry, and tracing paths (`studies/agent-harness-study/sources/crewai/lib/crewai/src/crewai/events/types/observation_events.py:46-131`; `studies/agent-harness-study/sources/crewai/lib/crewai/src/crewai/events/event_listener.py:544-622`; `studies/agent-harness-study/sources/crewai/lib/crewai/src/crewai/events/listeners/tracing/trace_listener.py:500-544`).

## Notable Patterns

- A planner-to-todo compiler creates an executable representation without changing the original task description (`studies/agent-harness-study/sources/crewai/lib/crewai/src/crewai/experimental/agent_executor.py:314-372`).
- Todo transitions are the lifecycle boundary for plan-step events, so execution state and observability are updated together (`studies/agent-harness-study/sources/crewai/lib/crewai/src/crewai/experimental/agent_executor.py:374-450`).
- Replanning is incremental: only pending todos are replaced, while prior completed, failed, and running records remain available as context (`studies/agent-harness-study/sources/crewai/lib/crewai/src/crewai/experimental/agent_executor.py:2504-2589`; `studies/agent-harness-study/sources/crewai/lib/crewai/src/crewai/utilities/planning_types.py:185-195`).
- Parallel ready todos use gathered execution with per-result exception handling and order-preserving mapping (`studies/agent-harness-study/sources/crewai/lib/crewai/src/crewai/experimental/agent_executor.py:1185-1319`).
- Plan-step identifiers propagate into tool-usage payloads, enabling correlation between plan state and execution telemetry (`studies/agent-harness-study/sources/crewai/lib/crewai/src/crewai/agents/step_executor.py:384-385`, `:422-423`, `:438-439`).
- The test suite exercises both normal planning and failure-oriented behavior, including malformed planner/observer output and failed dependencies (`studies/agent-harness-study/sources/crewai/lib/crewai/tests/utilities/test_structured_planning.py:575-668`; `studies/agent-harness-study/sources/crewai/lib/crewai/tests/agents/test_agent_executor.py:1989-2346`).

## Tradeoffs

- **Centralized mutation versus agent autonomy.** Executor-only updates make plan mutation auditable and prevent arbitrary plan corruption, but no direct LLM-callable update operation was found in the planner contract (`studies/agent-harness-study/sources/crewai/lib/crewai/src/crewai/utilities/reasoning_handler.py:50-104`; `studies/agent-harness-study/sources/crewai/lib/crewai/src/crewai/experimental/agent_executor.py:2504-2589`).
- **Rich adaptation versus bounded cost.** Observation and replanning improve recovery, while refinement attempts, step iterations, timeouts, and replan count cap model calls and execution duration (`studies/agent-harness-study/sources/crewai/lib/crewai/src/crewai/agent/planning_config.py:98-142`; `studies/agent-harness-study/sources/crewai/lib/crewai/src/crewai/experimental/agent_executor.py:986-1016`).
- **Structured decisions versus heuristic detection.** Structured observation fields drive many paths, but dynamic replanning also searches the last message for fixed indicators (`studies/agent-harness-study/sources/crewai/lib/crewai/src/crewai/experimental/agent_executor.py:2452-2502`). This can miss semantically equivalent failure signals or react to incidental wording.
- **Availability versus correctness on observer failure.** If observer execution raises, the fallback marks the step successful and keeps the plan, allowing progress but risking continuation after an incorrectly assessed step (`studies/agent-harness-study/sources/crewai/lib/crewai/src/crewai/agents/planner_observer.py:191-213`; `studies/agent-harness-study/sources/crewai/lib/crewai/tests/agents/test_agent_executor.py:1216-1247`).
- **Checkpoint compatibility versus durable planning storage.** Flow checkpointing can carry executor state across a resume, but the inspected source provides no evidence of a planner-specific durable store or a planner-specific step-level persistence protocol (`studies/agent-harness-study/sources/crewai/lib/crewai/src/crewai/agent/core.py:1573-1603`; `studies/agent-harness-study/sources/crewai/lib/crewai/src/crewai/experimental/agent_executor.py:126-161`).
- **Modern structure versus legacy compatibility.** The legacy crew-level planner remains supported but uses a free-form plan injected into task text, which gives it weaker execution constraints than the modern todo path (`studies/agent-harness-study/sources/crewai/lib/crewai/src/crewai/utilities/planning_handler.py:15-164`; `studies/agent-harness-study/sources/crewai/lib/crewai/src/crewai/crew.py:1427-1453`).

## Failure Modes / Edge Cases

- Malformed planner step fields are defaulted, and function-call JSON failures can fall back to text parsing (`studies/agent-harness-study/sources/crewai/lib/crewai/src/crewai/utilities/reasoning_handler.py:386-444`). This avoids immediate failure but may create underspecified steps.
- An unparseable observation becomes a conservative unsuccessful observation that invalidates the remaining plan (`studies/agent-harness-study/sources/crewai/lib/crewai/src/crewai/agents/planner_observer.py:342-352`).
- An observer LLM exception takes the opposite, availability-oriented path: it emits a failure event but assumes success and continues (`studies/agent-harness-study/sources/crewai/lib/crewai/src/crewai/agents/planner_observer.py:191-213`).
- Missing required tool usage fails the step, while an optional tool requirement is allowed (`studies/agent-harness-study/sources/crewai/lib/crewai/src/crewai/agents/step_executor.py:504-512`).
- Failed dependencies are terminal for readiness, allowing downstream progress; this may be useful for partial completion but can produce execution that proceeds despite an incomplete prerequisite (`studies/agent-harness-study/sources/crewai/lib/crewai/src/crewai/utilities/planning_types.py:112-119`; `studies/agent-harness-study/sources/crewai/lib/crewai/tests/agents/test_agent_executor.py:2308-2346`).
- No ready todo while the plan is incomplete is treated as a replan condition rather than successful completion (`studies/agent-harness-study/sources/crewai/lib/crewai/src/crewai/experimental/agent_executor.py:1049-1060`).
- Replan failure keeps the previous todos and records the failure in the last replan reason; reaching the replan limit finalizes with current results (`studies/agent-harness-study/sources/crewai/lib/crewai/src/crewai/experimental/agent_executor.py:2577-2589`, `:986-992`).
- A step can hit its iteration or timeout limit, and the overall executor can route to a final answer after the maximum iteration boundary (`studies/agent-harness-study/sources/crewai/lib/crewai/src/crewai/agents/step_executor.py:336-340`, `:546-554`; `studies/agent-harness-study/sources/crewai/lib/crewai/tests/agents/test_agent_executor.py:2123-2155`).
- Reinvocation resets executor state, so a new run does not accidentally reuse prior plan execution state (`studies/agent-harness-study/sources/crewai/lib/crewai/tests/agents/test_agent_executor.py:2177-2201`).
