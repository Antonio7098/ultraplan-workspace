## 03.08 — Subagent and Forked-Loop Design

### Source Info

| Field | Value |
|-------|-------|
| Name | crewai |
| Path | `studies/agent-harness-study/sources/crewai` |
| Language / Stack | Python (Pydantic v2, asyncio + thread-pool hybrid), agent framework built on top of `liteLLM`/`BaseLLM` and `a2a-sdk` for cross-process delegation |
| Analyzed | 2026-07-17 |

## Summary

CrewAI exposes **two distinct delegation mechanisms** with meaningfully different semantics:

1. **In-process delegation as tool calls** via `AgentTools` in `lib/crewai/src/crewai/tools/agent_tools/agent_tools.py:16-36` — when `allow_delegation=True` and a crew has more than one agent, the crew injects a `DelegateWorkTool` (`lib/crewai/src/crewai/tools/agent_tools/delegate_work_tool.py:16-30`) and an `AskQuestionTool` (`lib/crewai/src/crewai/tools/agent_tools/ask_question_tool.py:14-28`) into the current agent's tool list. Both tools call `BaseAgentTool._execute` (`lib/crewai/src/crewai/tools/agent_tools/base_agent_tools.py:46-124`) which picks a coworker by role match and calls `selected_agent.execute_task(task_with_assigned_agent, context)` (line 120). The child agent runs **in the same process** with its own `agent_executor`, its own memory retrieval (`base_agent_tools.py:112-116`), and emits its own `AgentExecutionStartedEvent`/`AgentExecutionCompletedEvent`. Errors during the child run are converted to a localized error string returned through `agent_tool_execution_error` (`base_agent_tools.py:121-124`) — they never re-raise to the caller, so the parent agent treats delegation failure as a recoverable string reply. Delegation is tracked as a `delegations` counter on `Task` (`lib/crewai/src/crewai/task.py:1067-1071`) by `track_delegation_if_needed` (`lib/crewai/src/crewai/utilities/agent_utils.py:1174-1188`) — recognizing the names `"Delegate work to coworker"` and `"Ask question to coworker"` (declared in `DELEGATION_TOOL_NAMES` at `agent_utils.py:1166-1171`).

2. **Cross-process A2A delegation** via `wrap_agent_with_a2a_instance` (`lib/crewai/src/crewai/a2a/wrapper.py:97-237`) — when an `Agent` has an `a2a` field set, the generic `AgentMeta` metaclass (`lib/crewai/src/crewai/agent/internal/meta.py:14-81`) automatically monkey-patches `execute_task`, `aexecute_task`, `kickoff`, and `kickoff_async` on the instance (`wrapper.py:118-235`) to route through `_execute_task_with_a2a` → `_delegate_to_a2a` (`wrapper.py:1231-1418`). The wrapper drives a **multi-turn conversation loop** (`wrapper.py:1272-1407`) up to `max_turns` (default 10, `config.py:491-493`) where the parent agent produces a structured `AgentResponseProtocol` (`lib/crewai/src/crewai/a2a/types.py`) with `is_a2a=True` and an `a2a_ids` list to choose the remote agent. Messages are dispatched through `execute_a2a_delegation` (`lib/crewai/src/crewai/a2a/utils/delegation.py:135-236`) over JSON-RPC / gRPC / HTTP+JSON negotiated in `_aexecute_a2a_delegation_impl` (`delegation.py:375-653`) with `context_id`/`task_id`/`reference_task_ids` lineage carried on the wire. The same wrapper also injects an A2A-server surface onto the agent via `inject_a2a_server_methods` (`lib/crewai/src/crewai/a2a/utils/agent_card.py:579-596`), giving it `agent.to_agent_card(url)` (`agent_card.py:485-596`) so any `Agent`/`Crew` can be exposed as a remote peer. LiteAgent reuses the same wrapper through a model validator (`lib/crewai/src/crewai/lite_agent.py:316-367`).

Trace nesting is **explicit and central**: the event bus (`lib/crewai/src/crewai/events/event_bus.py:540-568`) assigns every emitted event a `parent_event_id`, `triggered_by_event_id`, `previous_event_id`, `emission_sequence` via a `contextvars`-backed scope stack (`lib/crewai/src/crewai/events/event_context.py:41-71`). `EventRecord` (`lib/crewai/src/crewai/state/event_record.py:99-225`) wires bidirectional `parent`/`child`/`trigger`/`triggered_by`/`next`/`previous`/`started`/`completed_by` edges automatically (`event_record.py:123-144`) and traverses them with `descendants(event_id)` for full causal trees (`event_record.py:160-191`).

However, the model has **clear limits**:
- In-process delegation is **strictly sequential**: `DelegateWorkTool._execute` is awaited synchronously inside the tool call, which is itself one LLM tool step of the parent agent (`base_agent_tools.py:120`). A single LLM step can fire only one delegation at a time. There is no batched parallel coworker call.
- A2A delegation is **one endpoint per turn**. `_prepare_delegation_context` (`wrapper.py:1014-1063`) picks `agent_id = agent_response.a2a_ids[0]` and the turn loop (`wrapper.py:1272-1407`) calls `execute_a2a_delegation` once per iteration on that single endpoint. Multiple endpoints are gathered for *card fetching only* (`wrapper.py:262-299` and `wrapper.py:1436-1460` both fan out via `concurrent.futures.ThreadPoolExecutor` / `asyncio.gather`), not for actual delegation.
- `A2AParallelDelegationStartedEvent` / `A2AParallelDelegationCompletedEvent` exist (`lib/crewai/src/crewai/events/types/a2a_events.py:631-661`) and are registered as `SCOPE_STARTING_EVENTS`/`SCOPE_ENDING_EVENTS`/`VALID_EVENT_PAIRS` (`event_context.py:264`, `:315`, `:366`) — but **no production code path emits them** (`grep` for `A2AParallelDelegationStartedEvent\(` returns only the class definition). This is a planned-but-unfinished scaffolding: declared surface, no implementation.
- An `_parallel_delegation: bool = PrivateAttr(default=False)` is reserved on both `A2AConfig` and `A2AClientConfig` (`config.py:444`, `config.py:541`), again unread elsewhere.

Memory is **shared, not isolated**, in the in-process path: `BaseAgent._retrieve_memory_context` (`lib/crewai/src/crewai/agent/core.py:583-646`) reads `getattr(self, "memory", None) or (getattr(self.crew, "_memory", None) if self.crew else None)` (line 610-612), so a delegated child agent sees the same memory scope as the parent crew. There is no per-call memory copy. For A2A, memory is *transferred by copy of the prompt and final reply text only* — the wire format (`delegation.py:526-564`) carries `task_description`, `context`, `input_files`, and explicit `extensions` metadata, but no CrewAI memory store IDs.

## Rating

**7 / 10** — Clear model with two complementary delegation paths, tested interfaces, and observable trace nesting via the event bus + `EventRecord` graph, but parallel multi-endpoint delegation exists in name only, child state is mostly inherited rather than isolated, and synchronous in-process delegation cannot recover beyond a localized string error.

## Answers to Dimension Questions

1. **Can agents delegate?**
   Yes, through two distinct paths. In-process via `AgentTools` (delegation/ask tools on the LLM tool surface) — `lib/crewai/src/crewai/tools/agent_tools/agent_tools.py:16-36`. Cross-process via the A2A protocol, achieved by setting `Agent.a2a` (single config or list) — `lib/crewai/src/crewai/agent/core.py:321-333`; the metaclass activates the wrapper at `lib/crewai/src/crewai/agent/internal/meta.py:48-71`.

2. **Is delegation just a tool call or a real child run?**
   In-process: a real child run, but only one per LLM step. `BaseAgentTool._execute` calls `selected_agent.execute_task(task_with_assigned_agent, context)` synchronously inside the parent's tool loop — `lib/crewai/src/crewai/tools/agent_tools/base_agent_tools.py:120`. The child builds its own `Task`, fires `AgentExecutionStartedEvent` (`lib/crewai/src/crewai/agent/core.py:824-832`), invokes its own `agent_executor`, and reuses its own memory scope. A2A: a real remote run on a separate process; `_delegate_to_a2a` performs a `Message` POST to the remote `AgentCard` endpoint and consumes the streamed `TaskStateResult` — `lib/crewai/src/crewai/a2a/utils/delegation.py:135-653`.

3. **Is child state isolated?**
   In-process: **no**. `Agent` runs its own executor but shares the same `crew._memory` if its own `memory` is unset (`lib/crewai/src/crewai/agent/core.py:609-620`). The `_add_delegation_tools` step uses `agents_for_delegation = [agent for agent in self.agents if agent != task.agent]` — the *same* agent instances are reused (`lib/crewai/src/crewai/crew.py:1794`). The child's tool list is freshly built per `get_delegation_tools` call, but the underlying `BaseAgent` (and thus its `agent_executor` cache, `_last_messages`) is the long-lived instance. A2A: **yes at the wire level** — only `task_description`, `context`, `input_files`, `extensions`, and `metadata` cross the boundary (`lib/crewai/src/crewai/a2a/utils/delegation.py:526-564`); the remote CrewAI instance is freshly loaded by the server-side handler.

4. **Are child traces linked to parent traces?**
   Yes, via the event bus scope stack. Every event picks up `parent_event_id` from `get_current_parent_id()` (`lib/crewai/src/crewai/events/event_bus.py:561-564`) and is stored as a node in `EventRecord` with a `parent`/`child` edge (`lib/crewai/src/crewai/state/event_record.py:123-145`). `descendants(event_id)` produces the full sub-tree (`event_record.py:160-191`). A2A events are scoped — `a2a_delegation_started`/`completed`, `a2a_conversation_started`/`completed`, `a2a_server_task_started`/`completed` are registered in `SCOPE_STARTING_EVENTS`/`SCOPE_ENDING_EVENTS` and `VALID_EVENT_PAIRS` (`lib/crewai/src/crewai/events/event_context.py:240-369`). The trace listener picks them up and submits them as `TraceEvent` with `parent_event_id` from the live scope (`lib/crewai/src/crewai/events/listeners/tracing/trace_listener.py:921-935`).

5. **Can child loops run in parallel?**
   Not through the delegation surface itself. Multi-endpoint **fetch** of agent cards runs concurrently (`lib/crewai/src/crewai/a2a/wrapper.py:262-299` thread pool, `:1436-1460` `asyncio.gather`), but the delegation loop drives **one endpoint per turn** (`lib/crewai/src/crewai/a2a/wrapper.py:1272-1307`). Two abandonment paths exist: declared scope events for `A2AParallelDelegationStartedEvent`/`A2AParallelDelegationCompletedEvent` that are never emitted (`lib/crewai/src/crewai/events/types/a2a_events.py:631-661`) and an unused `_parallel_delegation` PrivateAttr (`lib/crewai/src/crewai/a2a/config.py:444`, `:541`). The closest thing to "parallel subagents" is `kickoff_for_each_async`, which runs **independent crews** (whole-graph copies) in parallel via `asyncio.gather` over `crew.copy()` instances (`lib/crewai/src/crewai/crews/utils.py:440-504`). These are not subagents of one parent — they are sibling crews with no causal link through the event record.

## Architectural Decisions

- **Tool-call-first delegation model.** Delegation is modeled as an `AgentTools` instance whose two `BaseTool` subclasses (`DelegateWorkTool`, `AskQuestionTool`) are injected into the agent's tool schema when `allow_delegation=True`. This keeps delegation *opt-in per call* — only when the LLM chooses to call the tool does a delegation actually happen — at the cost of forcing the parent through another full LLM step per delegation.
- **Dynamic method patching for A2A.** Rather than subclass `Agent`, the framework uses `AgentMeta` (`lib/crewai/src/crewai/agent/internal/meta.py`) to install `wrap_agent_with_a2a_instance(self, ...)` after `post_init_setup` runs if `self.a2a is not None`. The wrapper swaps `execute_task`, `aexecute_task`, `kickoff`, and `kickoff_async` on the *instance* via `object.__setattr__(agent, name, MethodType(fn, agent))` (`lib/crewai/src/crewai/a2a/wrapper.py:166-235`). The substituted method consults `self.a2a` and either delegates or falls through to the original. This is fragile to Pydantic's permission model but avoids class explosion.
- **Multi-turn turn loop with bounded max_turns.** `A2AClientConfig.max_turns` defaults to 10 (`lib/crewai/src/crewai/a2a/config.py:491-493`); the wrapper enforces a `for turn_num in range(ctx.max_turns)` loop (`lib/crewai/src/crewai/a2a/wrapper.py:1272`). Each turn ends in either a `TaskState.completed` (with `trust_remote_completion_status` optionally short-circuiting the rest of the loop — `wrapper.py:1066-1143`) or `TaskState.input_required` returned by the remote agent.
- **Structured prompt augmentation, not function-calling for A2A selection.** The parent agent is given a `task.description` rewritten by `_augment_prompt_with_a2a` (`lib/crewai/src/crewai/a2a/wrapper.py:639-743`) to inject available agent cards and conversation history, and is forced to emit an `AgentResponseProtocol` (a Pydantic model with `is_a2a`, `a2a_ids`, `message`) so the wrapper can choose exactly one endpoint. This decouples LLM provider specifics from the delegation loop.
- **Trace correlation via contextvars.** Rather than threading `parent_event_id` through arguments, the event bus uses `contextvars.ContextVar` for an event-id stack (`lib/crewai/src/crewai/events/event_context.py:41-71`) and walks that stack on every emit (`event_bus.py:544-564`). This works for sync code, async code, and `concurrent.futures.ThreadPoolExecutor` snapshots via `contextvars.copy_context()` (`lib/crewai/src/crewai/a2a/wrapper.py:233-235` and `:283`).
- **Event graph with 8 edge types.** `EventRecord` models causality with `parent`/`child` (scope nesting), `trigger`/`triggered_by` (explicit causal pointer), `next`/`previous` (linear sequence), and `started`/`completed_by` (lifecycle pairs) (`lib/crewai/src/crewai/state/event_record.py:52-61`, `:110-144`). This is overkill for plain subagent delegation but cheap because wiring is automatic.
- **Delegation failures return as strings.** When a delegated child raises, `BaseAgentTool._execute` swallows the exception into a localized error message via `I18N_DEFAULT.errors("agent_tool_execution_error")` (`lib/crewai/src/crewai/tools/agent_tools/base_agent_tools.py:121-124`). This prevents one bad coworker from crashing the parent agent's task, at the cost of making it hard to programmatically distinguish "child returned nothing" from "child exploded".

## Notable Patterns

- **Tool-as-coworker.** `DelegateWorkTool` is an `AgentTool`-style wrapper that resolves a coworker by sanitized role match and forwards a `Task` to `selected_agent.execute_task`. Reuses the agent's own execution machinery (`AgentExecutor`/`CrewAgentExecutor`) so the child benefits from the parent's tools/instructions — `lib/crewai/src/crewai/tools/agent_tools/base_agent_tools.py:61-124`.
- **Method-type monkey-patching.** A2A wrapper rebinds instance methods via `MethodType` after model construction. The wrapper preserves a reference to the original via `original_execute_task = agent.execute_task.__func__` and re-invokes it after prompt augmentation (`lib/crewai/src/crewai/a2a/wrapper.py:115-150`).
- **Trust-flag early-return on multi-turn.** `trust_remote_completion_status=True` collapses the multi-turn loop on the first `TaskState.completed` reply, returning the remote result verbatim (`lib/crewai/src/crewai/a2a/wrapper.py:1066-1142`). Useful for fire-and-forget delegation.
- **Reference-task lineage across A2A turns.** On `TaskState.completed`, the completed `task_id` is appended to `task.config["reference_task_ids"]` (`wrapper.py:1115-1120`), so subsequent turns carry a chain of past task IDs into `_delegate_to_a2a`'s `metadata` and on into the next `Message`. This is the closest the framework gets to a structured subsession ID.
- **Role-sanitization tolerant of malformed LLM JSON.** `sanitize_agent_name` collapses whitespace, lowercases, and strips quotes so a truncated JSON coworker reference like `"research` still matches `researcher` (`lib/crewai/src/crewai/tools/agent_tools/base_agent_tools.py:20-35`).
- **Hook-into-event-bus everywhere.** Both delegation paths emit their start/complete pairs as Pydantic events on `crewai_event_bus`, registered on the listener side at `lib/crewai/src/crewai/events/listeners/tracing/trace_listener.py:600-740`. Listeners (`@event_bus.on(...)`) are open to extension — the trace listener auto-skips claims when a flow already owns the batch (`trace_listener.py:770-781`).

## Tradeoffs

- **Sequential delegation by default.** One LLM step can only fire one delegation tool; there is no `DelegateWorkTool.__call__` that fans out to multiple coworkers concurrently. For multi-agent synthesis the user pays N LLM steps and N serial child runs.
- **Shared mutable in-process state.** Because the child is the *same* `BaseAgent` instance as the one in the crew, the delegated run inherits the agent's `_last_messages`, `cache_handler`, and `_rpm_controller`. A child that mutates `_last_messages` (via `save_last_messages` in `core.py:680`) can influence a later parent turn.
- **Sibling vs parent-child ambiguity.** `kickoff_for_each_async` runs N **independent crews** in parallel, but they share no event scope and produce no parent-child trace link. Conflating "parallel subagents" with "parallel crews" obscures the accountability graph.
- **Parallel delegation is half-built.** The events and the PrivateAttr are in place; the loop is not. A reader of `event_context.py:264`, `:315`, `:366` would reasonably conclude that parallel delegation is supported — and would then find no production code path that emits the events.
- **Error masking in-process.** The synchronous string-conversion of exceptions (`base_agent_tools.py:121-124`) is friendly to multi-agent recovery but loses stack traces. A2A propagation in contrast raises up via `execute_a2a_delegation` and surfaces as `A2AConnectionErrorEvent` / `A2AAuthenticationFailedEvent` (`lib/crewai/src/crewai/a2a/utils/delegation.py:335-350`, `events/types/a2a_events.py:478-498`).
- **Metaclass dependency.** `AgentMeta` mutates instances after model construction. Pydantic v2's behavior around `model_config` and `__init_subclass__` collisions is a known risk surface that is muted here by wrapping only when `a2a is not None`.

## Failure Modes / Edge Cases

- **Unknown coworker role.** `BaseAgentTool._execute` falls back to `agent_tool_unexisting_coworker` with a list of available roles (`lib/crewai/src/crewai/tools/agent_tools/base_agent_tools.py:88-108`). Parent agent sees the error as a tool result string.
- **Child exception.** In-process, swallowed and reported as `agent_tool_execution_error` (`base_agent_tools.py:121-124`); the parent's tool loop continues. A2A, exception bubbles via `execute_a2a_delegation` and triggers `A2ADelegationCompletedEvent(status="failed")` (`lib/crewai/src/crewai/a2a/utils/delegation.py:335-370`); on a hard parse error the wrapper's `_emit_delegation_failed` returns a string (`lib/crewai/src/crewai/a2a/wrapper.py:822-846`).
- **Conversation overrun.** `_handle_max_turns_exceeded` (`lib/crewai/src/crewai/a2a/wrapper.py:761-819`) emits a `failed` event and raises `Exception("A2A conversation exceeded maximum turns ({max_turns})")` if the remote agent never flips `is_a2a=false`.
- **Card fetch with `fail_fast=True`.** `_fetch_card_from_config` raises `RuntimeError("Failed to fetch agent card from {endpoint}")` in the thread-pool path (`lib/crewai/src/crewai/a2a/wrapper.py:289-294`) and `_afetch_card_from_config` re-raises in async (`wrapper.py:1451-1455`).
- **Sync delegation inside a running event loop.** `execute_a2a_delegation` detects a running loop and dispatches through `ThreadPoolExecutor(max_workers=1)` with `contextvars.copy_context()` to avoid `asyncio.run` from crashing (`lib/crewai/src/crewai/a2a/utils/delegation.py:226-236`).
- **gRPC transport rejected with HTTP-digest auth.** `_create_grpc_channel_factory` raises `ValueError("HTTPDigestAuth is not supported with gRPC transport.")` (`lib/crewai/src/crewai/a2a/utils/delegation.py:782-788`).
- **Missing agent executor.** `Agent._execute_without_timeout` raises `RuntimeError("Agent executor is not initialized.")` (`lib/crewai/src/crewai/agent/core.py:903`), and the async variant checks the same (`core.py:1025`).
- **Memory scope mismatch.** Delegated child reads from `crew._memory` if no per-agent memory is configured (`lib/crewai/src/crewai/agent/core.py:609-620`), silently coupling the child's recall to the crew's storage. Callers expecting isolation are surprised.
- **Token-load regression in delegation reuse.** Delegation consumes the parent's tokens (the child uses `selected_agent`'s LLM via `agent_executor.ainvoke`), but the parent's `_rpm_controller` is shared (`core.py:1068-1071`), so deep delegation trees can self-throttle.

## Future Considerations

- **Implement parallel A2A.** The events and PrivateAttr hint at a fan-out where one parent prompt chooses multiple `a2a_ids` and dispatches in parallel. Recommended path: add a `_afan_out_delegation(agent_ids, ...)` that calls `aiohttp`-backed `aexecute_a2a_delegation` via `asyncio.gather`, persists each branch as `A2AParallelDelegationStartedEvent`/`A2AParallelDelegationCompletedEvent` sub-scopes, then resumes `_handle_agent_response_and_continue` with merged transcripts. Existing scope pairing (`event_context.py:264`, `:315`, `:366`) is already in place.
- **Per-call child isolation.** When delegating in-process, snapshot `agent._last_messages` and `_cache_handler` before handing control to the child and restore on return, so the parent does not inherit the child's last-message buffer (`lib/crewai/src/crewai/agent/core.py:202-203`, `:680`).
- **Propagate a Subsession ID over A2A.** A2A wire already carries `context_id` and `task_id`. Combining them with the parent's `flow_id`/`crew_id` would let a remote viewer reconstruct parent-child causation without depending on the server side being CrewAI.
- **Real parallel delegation tools.** A `DelegateWorkTool` variant that takes a list of `coworker`s and synchronizes via `asyncio.gather` (or a `ThreadPoolExecutor` bounded to `min(8, len(coworkers))`, mirroring `experimental/agent_executor.py:1714`) would let multi-agent synthesis happen in one LLM turn. Each child should still emit its own scoped `agent_execution_started`/`completed` pair so the trace graph captures the fan-out.
- **Surface delegation exceptions upstream.** Optional `return_exceptions=True` policy on the in-process delegation tool so callers can choose between friendly-string and structured-error responses.
- **Remove the `_parallel_delegation` dead flag** until it's wired up — it currently misleads readers about capability parity.

## Questions / Gaps

- **Memory scope causality across delegation.** No explicit mechanism prevents a delegated child from mutating the parent's conversation history beyond `save_last_messages`. There is no test that asserts a delegated child's message buffer is independent.
- **Trace completeness for parallel crews.** `crew.kickoff_for_each_async` creates `crew.copy()` instances that each spawn their own event scope, but no parent-child edge is emitted between them. From an `EventRecord.roots()` perspective, N independent crews look identical to N truly independent runs.
- **Concurrency safety of `_execute_with_timeout`.** The synchronous path uses `concurrent.futures.ThreadPoolExecutor` with `contextvars.copy_context()` (`lib/crewai/src/crewai/agent/core.py:872-890`); if the LLM call also internally spins up a thread, nested context propagation may diverge from the parent's stack. No test exists for this.
- **A2A max-turns enforcement semantics.** When `is_a2a` stays `True` through every turn, `_handle_max_turns_exceeded` raises — but the parent agent has no way to retry with a higher max_turns from the response model itself. Useful error code?
- **Hierarchical process delegation.** The hierarchical manager uses `_create_manager_agent` with `AgentTools(agents=self.agents).tools()` and `allow_delegation=True` (`lib/crewai/src/crewai/crew.py:1509-1518`) but the manager is restricted from holding its own tools (`crew.py:1498-1505`). The interaction between the manager's delegation tools and the underlying task agents' tools is not deeply tested for graph correctness.
- **Speculative test gaps.** `lib/crewai/tests/agents/test_a2a_trust_completion_status.py` is an integration test; there is no unit test that exercises the new `_parallel_delegation` flag or the parallel events — confirming (by absence of test) that those events are surface-only today.
